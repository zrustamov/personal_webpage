How My Systems Work

This page explains the architecture choices behind my AI systems, from retrieval to orchestration and answer generation.

RAG Pipeline

Query intake, text normalization, candidate generation, and grounded response synthesis are split into measurable stages for reliability.

Hybrid Retrieval

Dense retrieval captures semantics while sparse retrieval protects exact legal terms and entities. Fusion combines both for stronger recall and precision.

Multi-Agent Design

Specialized agents handle intent, routing, and response strategy so the system remains modular, testable, and extensible.

Typical Flow

1. User query and intent classification
2. Query normalization and legal scope filtering
3. Dense + sparse retrieval from Milvus and index stores
4. Reciprocal Rank Fusion + LLM reranking
5. Evidence-grounded response generation
